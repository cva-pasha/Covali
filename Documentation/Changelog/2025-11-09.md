# Changelog - November 9, 2025

### November 9, 2025, 12:50

### Summary
Added method overload to AddEventSourcing to accept Type[] parameters, improving API convenience by allowing type arrays instead of requiring assembly arrays. Version bumped to 9.1.3.

### Statistics

| Metric | Value |
|--------|---------|
| Files modified | 2 |

### Affected Modules

- Covali.EventSourcing.Extensions

### Technical Decisions

**API Overload Addition:**
- Added new `AddEventSourcing(IServiceCollection, ServiceLifetime, params Type[])` overload
- Internally converts types to assemblies via `types.Select(t => t.Assembly).ToArray()`
- Original method now accepts `Assembly[]` instead of `Type[]` for clarity
- Maintains backward compatibility while offering more intuitive API surface

**Before:**
```csharp
public static IServiceCollection AddEventSourcing(
    this IServiceCollection services,
    ServiceLifetime lifetime,
    params Type[] types
)
{
    // Implementation extracted assemblies from types internally
}
```

**After:**
```csharp
// New overload for Type[] - convenience method
public static IServiceCollection AddEventSourcing(
    this IServiceCollection services,
    ServiceLifetime lifetime,
    params Type[] types
)
{
    return AddEventSourcing(services, lifetime, types.Select(t => t.Assembly).ToArray());
}

// Original method renamed parameter for clarity
public static IServiceCollection AddEventSourcing(
    this IServiceCollection services,
    ServiceLifetime lifetime,
    params Assembly[] assemblies
)
{
    // Implementation uses assemblies directly
}
```

### Benefits

✅ **API Clarity:** Separate overloads make intent explicit - use types when convenient, assemblies when precise
✅ **Backward Compatibility:** Existing code using Type[] continues to work without changes
✅ **Flexibility:** Developers can choose the most appropriate overload for their use case
